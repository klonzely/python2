## Описание проекта

В данной лабораторной работе разработан пользовательский класс на языке Python, моделирующий сущность **Player** (игрок в ролевой игре). Реализация выполнена в трёх уровнях сложности (оценка 3, 4, 5), демонстрируя:

- инкапсуляцию (закрытые поля);
- конструктор с валидацией входных данных;
- свойства (`@property`) для доступа к атрибутам;
- магические методы `__str__`, `__repr__`, `__eq__`;
- бизнес-методы, изменяющие состояние объекта;
- атрибуты класса;
- вынесение валидации в отдельный модуль;
- внутреннее состояние объекта и зависящее от него поведение.

## Класс `Player`

### Атрибуты

| Атрибут       | Тип         | Описание                              |
|---------------|-------------|---------------------------------------|
| `name`        | `str`       | Имя персонажа (непустая строка).      |
| `level`       | `int`       | Уровень (от 1 до `MAX_LEVEL`).        |
| `health`      | `float`     | Здоровье (≥ 0).                       |
| `experience`  | `int`       | Опыт (≥ 0).                           |
| `is_alive`    | `bool`      | Состояние (жив/мёртв, только чтение). |

Атрибут класса:
- `MAX_LEVEL = 60` – максимально допустимый уровень.

### Методы

#### Конструктор `__init__(name, level, health, experience)`
Инициализирует объект с проверкой всех аргументов через сеттеры.

#### Свойства (геттеры и сеттеры)
- `name` – геттер и сеттер с валидацией (непустая строка).
- `level` – геттер и сеттер (целое число, от 1 до `MAX_LEVEL`, только для живого игрока).
- `health` – геттер и сеттер (число ≥ 0, при обнулении автоматически меняет состояние на "мёртв").
- `experience` – геттер и сеттер (целое ≥ 0, только для живого игрока).
- `is_alive` – геттер (состояние только для чтения).

#### Бизнес-методы
- `take_damage(damage: float)` – уменьшает здоровье на указанную величину (не ниже 0). Если здоровье становится 0, игрок умирает.
- `gain_experience(amount: int)` – добавляет опыт. При достижении порога (1000 × текущий уровень) уровень повышается на 1 (до `MAX_LEVEL`). Доступно только живому игроку.
- `revive()` – воскрешает мёртвого игрока, устанавливая здоровье в 50 и состояние в "жив".

#### Магические методы
- `__str__()` – удобочитаемое представление (для пользователей).
- `__repr__()` – однозначное представление (для разработчиков).
- `__eq__(other)` – сравнение двух игроков по всем атрибутам (включая состояние).

### Валидация (модуль `validate.py`)

Все проверки вынесены в отдельные функции:
- `validate_name(value)`
- `validate_level(value, max_level)`
- `validate_health(value)`
- `validate_experience(value)`
- `validate_damage(value)`

Каждая функция при несоответствии выбрасывает `TypeError` или `ValueError`.
