# Лабораторная работа №2: Реализация контейнера для класса `Weapon`

## Описание проекта

В данной лабораторной работе реализован контейнерный класс `WeaponCollection` (Арсенал) для хранения и управления объектами `Weapon` из ЛР-1. Реализует контейнер для группы оружия с возможностью добавления, удаления, поиска, сортировки и фильтрации.

## Краткое описание класса

`WeaponCollection` — это контейнерный класс для управления коллекцией оружия. Внутри хранит список объектов `Weapon`.

### Основные возможности:

- **Хранение** — список объектов `Weapon`
- **Управление** — добавление, удаление, доступ по индексу
- **Поиск** — по названию, урону, редкости
- **Сортировка** — по урону, названию, прочности, редкости
- **Фильтрация** — получение подколлекций (сломанное оружие, оружие с уроном выше порога, по редкости)
- **Итерация** — поддержка цикла `for`
- **Защита** — проверка типа, запрет дубликатов

## Методы коллекции

### Базовые операции

| Метод | Описание |
|-------|----------|
| `add(item)` | Добавить оружие |
| `remove(item)` | Удалить оружие |
| `remove_at(index)` | Удалить по индексу |
| `get_all()` | Получить список всего оружия |

### Поиск

| Метод | Описание |
|-------|----------|
| `find_by_name(name)` | Поиск по названию |
| `find_by_damage(damage)` | Поиск по урону |
| `find_by_rarity(rarity)` | Поиск по редкости |

### Магические методы

| Метод | Описание |
|-------|----------|
| `__len__` | Получение количества предметов |
| `__iter__` | Итерация по оружию |
| `__getitem__` | Доступ по индексу |
| `__contains__` | Проверка наличия оружия |

### Сортировка

| Метод | Описание |
|-------|----------|
| `sort_by_damage()` | Сортировка по урону |
| `sort_by_name()` | Сортировка по названию |
| `sort_by_durability()` | Сортировка по прочности |
| `sort(key)` | Универсальная сортировка |

### Фильтрация (возвращают новую коллекцию)

| Метод | Описание |
|-------|----------|
| `get_broken()` | Сломанное оружие |
| `get_by_min_damage(min_damage)` | Оружие с уроном не ниже порога |
| `get_by_rarity(rarity)` | Оружие указанной редкости |

## Демонстрация работы (`demo.py`)

### Сценарий 1 — Базовые операции

**Что демонстрируется:**
- создание пустой коллекции
- создание объектов `Weapon`
- добавление оружия в коллекцию
- вывод всего оружия
- удаление оружия
- проверка типа при добавлении

### Сценарий 2 — Поиск и проверка дубликатов

**Что демонстрируется:**
- защита от добавления дубликатов
- поиск оружия по названию
- поиск по урону
- поиск по редкости
- использование `len()` и итерации

### Сценарий 3 — Индексация и сортировка

**Что демонстрируется:**
- доступ по индексу через `__getitem__`
- сортировка по урону
- сортировка по названию
- сортировка по прочности
- удаление по индексу

### Сценарий 4 — Фильтрация коллекций

**Что демонстрируется:**
- получение сломанного оружия
- получение оружия с уроном выше порога
- получение оружия по редкости
- проверка, что фильтрация не изменяет исходную коллекцию

### Сценарий 5 — Бизнес-сценарии

**Что демонстрируется:**
- экипировка персонажа (выдача оружия из арсенала)
- ремонт сломанного оружия
- поиск самого мощного оружия
- формирование отчета с сортировкой по урону

## Примечания

- Класс `Weapon` должен быть реализован согласно первой лабораторной работе.
- Контейнер `WeaponCollection` полностью соответствует требованиям на оценки 3, 4, 5.
- Демонстрация включает все ключевые сценарии, подтверждающие работоспособность коллекции.
